# コントリビューションガイド

このプロジェクトへの貢献方法についてまとめています。ご一読のうえ、ご協力をお願いします。


## 貢献手段

エンジニア/非エンジニア問わず貢献いただけます。

### 非エンジニア向け

以下のような形で貢献することができます。

- 発見した不具合の報告
  [Issue](#issue課題管理) を作成することで不具合を報告できます
- 機能改善の提案
  上記と同じく [Issue](#issue課題管理) により提案できます

### エンジニア向け

- [Issue を作成](#issue-の作成方法) することができます
- リポジトリをフォークし、ご自身で機能追加や修正を行い [Pull Request](#pull-requestプルリクエスト) を送ることができます


## Issue（課題管理）

Issue とは、課題や問題点を管理するための機能です。
Issue を作成することで、開発者たちに向けて課題や問題点の共有を行うことができます。

自分で課題を修正できる場合はそのまま自分で修正してもよいですし、
難しければ課題作成のみ行ってほかの開発者に対応してもらうこともできます。

ただし、上げた課題が必ず対応される保証はできませんのでご注意ください。

### Issue の作成方法

タイトルは問題点をできるだけ具体的に表現する書き方にしてください。

**例**

```
NG: ヘッダーがおかしい
OK: ヘッダーのタイトルが枠からはみ出る場合がある
OK: ヘッダーのタイトルの position 値が正しくない

NG: スマートフォンでログインできない
OK: スマートフォン表示時にログインボタンをタップしても反応しない
OK: スマートフォン表示時のログインボタンが disabled になっている

NG: ダッシュボードのページでエラーになる
OK: ダッシュボードへアクセスすると 500 エラーと表示される
OK: ダッシュボードのデータフェッチ処理のエラーハンドリングが不足している
```

記載内容も、分かる範囲で構いませんのでできるだけ具体的にお願いいたします。


## Discussion（議論）

仕様に関する質問や、まだタスク化できない課題などは Discussion を利用してください。

**Discussion 例**
- 表示データはどこから取得しているか
- 例外処理の統一について
- デザインカンプがほしい


## Pull Request（プルリクエスト）

このリポジトリへの直接コミットは禁止です。
必ず fork して自分のリポジトリでブランチを作成し、プルリクエストを提出してください。

基本的には、作業前に issue を作成し、編集方針が定まってから着手してください。
ただし、ドキュメントの誤字修正や軽微なコード修正など、議論の必要が薄い場合は直接プルリクエストを作成しても構いません。

ブランチ命名規則はありませんので、自由なブランチ名でプルリクエストを作成していただいてよいです。
迷う場合は、参考として次のような規則を掲載しておきます（従う必要はありません）。

| 種類             | 命名例                | 説明                                               |
| :--------------- | :-------------------- | :------------------------------------------------- |
| issue対応        | issue/000             | issue に関連する作業（000 は issue 番号）          |
| ドキュメント     | docs/some-name        | ドキュメント関連の変更                             |
| 新機能           | feature/some-name     | 新機能の開発                                       |
| バグ修正         | fix/some-name         | 軽微なバグや機能修正（マイナスを 0 にする場合）    |
| 改善・リファクタ | enhancement/some-name | リファクタリングやエラー処理など（プラスする場合） |
